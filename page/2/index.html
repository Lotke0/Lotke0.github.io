buu-get_start_3dsctf_2016

发表于 2021-06-15 更新于 2021-06-16 分类于 buu ， pwn

分析

初探

main函数中存在明显的溢出漏洞，本题目又留出了后门函数，理论上来说通过栈溢出控制返回地址为后门函数，再在栈中放上指定的参数就能getflag。此方法在本地确实可行，但是远程会挂。。

更换思路

在ida中找到了mprotect()函数和read()函数

mprotect()函数可以用来修改一段指定内存区域的保护属性，所以可以用mprotect()函数修改一段内存属性为可读、可写、可执行，然后用read()向地址内写入shellcode，并返回到该地址即可getshell

解题

本题目要在执行完成mprotect()函数之后返回到read()函数，并且两个函数的参数都必须可控，所以要用到寄存器，在mprotect()函数执行完成后，返回到连续pop三个寄存器并返回的地址，就可以清理掉mprotect()执行的时候使用的参数，并且返回到read()，并且read的参数和返回地址可控。

简单了解一下 mprotect()

函数原型

mprotect()函数把自start开始的、长度为len的内存区的保护属性修改为prot指定的值。

prot可以取以下几个值，并且可以用“|”将几个属性合起来使用：

PROT_READ：表示内存段内的内容可写；

PROT_WRITE：表示内存段内的内容可读；

PROT_EXEC：表示内存段中的内容可执行；

PROT_NONE：表示内存段中的内容根本没法访问。

那么第三个参数怎么控制呢？

如果你打开/ usr / include / asm-generic，打开mman-common.h，你会发现这些值。

可读可写可执行对应的整数为: 0x1|0x2|0x3=00000001|00000010|00000100=00000111=7

然后进行逐步跟踪调试，发现运行结束后返回值为-1

buu-easyrop

发表于 2021-06-13 更新于 2021-06-14 分类于 buu ， pwn

程序分析

  1. main函数中生成4位的随机数写入变量buf中，传入函数inputcomp()（自行重命名）

  1. 函数inputcomp()将传入的随机数写入数组s，接收用户输入存入buf中，用strlen()读取buf字符串的长度，并以此长度用函数strncmp()比较字符串buf和s，若不相同则退出程序；若相同则返回buf的第八字节。

    此处要绕过字符串的比较，因为无法获取生成的随机数的值。只要将buf第一个字节设置为’\x00’（字符串终止符）即可让strlen()函数得出的长度为零，此时strncmp()比较两个字符串的前零位，恒相等。绕过比较后便可随意控制buf[7]的值

  2. 将函数inputcomp()返回的值传入main中最后一个函数中，如果传入的值不等于’\x7f’，系统便以传入的值为读取上限来读取用户输入并存入buf中。

    此处为溢出点

题解

思路

  1. 绕过字符串比较，控制buf[7]的值。
  2. 在第二个函数中进行溢出，构造rop链，通过puts函数泄露出read函数的地址并返回至main函数，从而泄露出libc基地址并为第二次溢出做准备。
  3. 重复步骤一
  4. 通过libc的基地址找出system和”/bin/sh”的地址，再次进行溢出，构造rop链，执行system(“/bin/sh”)

相关函数和构造参数详解

open()

open()的原型为

path，是包括路径的完整文件名，oflags是文件访问模式（即是什么方式打开文件，只读、只写还是可读并可写等），mode用于设定文件的访问权限。

open建立了一条到文件或设备的访问路径，如果调用成功，返回一个可以被read、write等其他系统调用的函数使用的文件描述符，而且这个文件描述是唯一的，不与任何其他运行中的进程共享，在失败时返回-1，并设置全局变量errno来指明失明的原因。

文件描述符

文件描述符是一些小数值，你可以通过它们访问的打开的文件设备，而有多少文件描述符可用取决于系统的配置情况。但是当一个程序开始运行时，它一般会有3个已经打开的文件描述符，就是

0：标准输入

1：标准输出

2：标准错误

那些数学（即0、1、2）就是文件描述符，因为在Linux上一切都是文件，所以标准输入（stdin），标准输出（stdout）和标准错误（stderr）也可看作文件来对待

read()

read()的原型为

read系统调用的作用是从与文件描述符相关的文件里读入nbytes个字节的数据，并把它们放到数据区buf中，返回读入的字节数，失败时返回-1。

当文件描述符为0时，从标准输入读取数据。

strncmp()

strncmp()函数原型为

C 库函数 int strncmp(const char *str1, const char *str2, size_t n) 把 str1 和 str2 进行比较，最多比较前 n 个字节。如果str1<str2，则返回值<0，如果str1>str2，则返回值>0，如果str1=str2，则返回值=0

system()

system()原型

如果无法启动shell运行命令，system将返回127；出现不能执行system调用的其他错误时返回-1。如果system能够顺利执行，返回那个命令的退出码。

32位传参

  1. 将要使用的参数从右向左依次压入栈
  2. 将返回地址压入栈
  3. 将bp压入栈
  4. 函数执行

若直接劫持栈中的返回地址N为某个函数的地址，该返回地址N后面的四位为该函数的返回地址M，返回地址M后面的数据才是该函数的参数。

buu_pwn5

发表于 2021-06-12 更新于 2021-06-14 分类于 buu ， pwn

查看文件信息&ida逆向

拖进ida查看主要逻辑

分析

逻辑

生成随机数，放入dword_804c044中，从标准输入读取用户输入，长度为99，并用printf函数输出，再次读取用户输入，将其与生成的随机数进行比较，若一致则getshell

漏洞

第一次读取用户输入后直接用printf进行输出，存在明显的格式化字符串漏洞。而且dword_804c044地址已知

做题思路

  1. 通过[padding]%p%p%p%p%p%p%p%p%p%p的方式确定printf在被调用时，格式化字符串为第几个参数

  2. 通过任意写将指定的数字写入dword_804c044,即可getshell

解题

  1. 确定printf在被调用时，格式化字符串为第几个参数

    确定格式化字符串为第十个参数

  2. 控制dword_804c044中的内容

    payload=”aaa”+”%12$n”+p32(0x0804C044)可以将dword_804c044中的内容控制为3
